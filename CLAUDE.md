# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

No build step, no lint, no npm install for the app — plain HTML/CSS/JS served as-is. The only exception is the browser E2E below, which is dev-only tooling; the app itself never needs a build or install.

```sh
# Run all tests (Node 18+) — the fast vm suite, install-free (gates deploy alongside e2e)
node --test

# Run a single test file
node --test tests/js/engine.test.js

# Run the game locally
open web/index.html   # works from file://, no network requests

# Browser E2E smoke test (one-time setup, then run)
npm install                       # installs @playwright/test (dev only)
npx playwright install chromium   # downloads the headless browser
npm run test:e2e                  # drives web/index.html in headless Chromium
```

Deploy is automatic: any push to `main` runs `.github/workflows/pages.yml` (tests gate the deploy), which publishes `web/` to the `gh-pages` branch. Live at https://soisanuk.github.io/last-baht-bus/.

**Testing cheats:** `CHEATS_ENABLED` (a `let` in engine-core.js — no build step, so the constant is the switch) gates hidden typed codes. **Currently `true`; the intended production default is `false`** — flip it before a real release. Codes are deliberately not surfaced (no autocomplete/decoration): `twoweekmillionaire` grants ฿2,000,000 for spending (handled early in `doCommand`, works in any state, costs no turn).

## Architecture

Zork-style text adventure in the Soi Sanuk / Pattaya universe. Same conventions as the trainer (`/Users/mario/thaicab`): **classic script tags sharing globals, no ES modules, no build step, works from `file://`**. The root `package.json` `"type": "module"` exists only so `node --test` treats test files as ESM. Load order in `index.html` matters (`thai → world → games → engine-core → engine-encounters → engine-play → engine-systems → engine-parser → …`); `main.js` loads last.

**Every player-facing option lives on three surfaces** — the typed parser (and its printed lists), the flyout wheel (`_kwActions`, term.js), and the input autocomplete (`engineComplete`/`_completePool`, engine.js). When adding, gating, or removing an option, change all three: put the rule in one engine-side helper (`_topicKnown`, `_travelDests`, `_playOptions`, `_c4Choices`, `_jpChoices`, `_gameVerbs`, `_salengItems` are the pattern) and have every surface consume it, then check `_kwActions` and `_completePool` for the affected verb before calling it done.

**Command-hint idiom** (what makes prose tappable): only ALL-CAPS *inside literal parentheses* decorates into a cmd kw — CAPS outside parens are plain emphasis, so put every actionable hint in parens and keep non-commands out of them (no CAPS room labels or stray `ON`/`BLACK` inside a paren group; note parens match across newlines). `<placeholders>` fold into the kw and make it an open prefill (`SEND <amount> TO <name>` → prefills `send `), even mid-phrase — never leaves orphan `TO`/`FOR` kws. `[x]` deliberately ends the capture so the base command fires as-is (`PLAY JACKPOT [bet]` → fires `play jackpot`). Modal screens must keep their options parenthesized and complete (`(FLIP 3 4 · FLIP 7)`, the checkout hotel list, `(NEW VACATION · MOVE TO PATTAYA)`), and modal state owns autocomplete (`_gameVerbs`, `G.pendingChoice` branches in `engineComplete`).

**The game is a staged sandbox.** `G.stage`: `act1` (the wallet quest, day 2 of a 7-day vacation) → `vacation` (Act One complete: `_checkAct1` fires once at `hotel_room`, gated in `_doGo` on `hasWallet`, scores the quest, converts score to happiness, and opens the room safe +฿3000) → `expat` (chosen at week's end; endless, +฿20000 savings; future home of the run-your-own-bar mechanic). `G.over` is never set; the long-game stat is `G.happy` (สนุก), fed by `_addHappy(n)` hooks across every system. Levels at 0/10/25/50/100; 100 = สบายสบาย, celebrated once via the `sabaiSabai` flag but non-terminal.

**The clock and the body.** Nights are `NIGHT_TURNS` (100) long, 10 turns/hour from 18:00; `_tick` accrues hunger/thirst, sobers you 1 drink per 20 turns, drains happiness when meters redline, and calls `_endNight(reason)` on collapse (hunger/thirst 100), blackout (drunk ≥ 9 via `_checkDrunk`), hospital (`G.hurt` ≥ 3), or dawn. `_endNight` resets dailies (hangover scales next evening's hunger/thirst), advances `G.day`, and respawns at `_hotelRoomId()` (or the beach pre-act1); day > 7 outside expat triggers `_endVacation` → `G.pendingChoice = "vacation_end"` gate → `_newVacation()` (happy resets, `bestHappy` tracks, no lead-in) or `_goExpat()`. Food/water via `FOOD_STALLS`/`_doEat`, plus `room.seven` (one street room per district) selling the cheese toastie/water/charger. BARFINE via `_doBarfine`: favor-gated, priced by `_barfinePrice` (×1.5 before 21:00, beer-bar fee waived after midnight except `POPULAR_GIRLS`, flash joints ×0.75); Soi 6 continues the night, elsewhere `_endNight("barfine")` +10 happy. The `freelancer` encounter (Beach Road/promenade) undercuts a barfine (฿700 solo / ฿1400 with Ning) but rolls her kind: mostly safe (+happy, threesome is the biggest single hit), else `_endNight("robbed")` lifts most of your remaining cash + any shades/Rolex — the canon "no mamasan, no recourse" downside. Not everyone is working: `bkktourist`/`jptourist` are non-transactional encounters where money is the *wrong* verb (it insults them, −happy); manners earn a free moment, and the Japanese traveller two-steps (offer re-arms `pendingEnc`) into a dancer threesome. `nightly:true` encounters restock each night — `_endNight` deletes all of them from `G.encDone`, not just `freelancer`. `britles` (British lesbian) and `punterwife` (Filipina wife) extend the non-transactional set: hands-on gets a confrontation, money gets a rebuff, and good vibes grant a **wingman** buff — `G.wingmanUntil = G.turns + WINGMAN_TURNS`, checked by `_wingman()`, worth +2 in `_favor` for its duration. `_maybeSelfBarfine` (favor ≥6, after midnight, once/night) sets `pendingEnc:"selfbf"`. Vacation stage: lobby ATM pays ฿3000 on first hotel exit per day (`G.atmDay`). Drunk ≥5 on lit streets rolls the repeatable `police` pseudo-encounter (30-turn cooldown via `G.lastPolice`); an adjacent `mamaTreat` bar's mamasan rescues at 70%.

Bar social life lives in `G.soc` (per-girl lady-drink counts, per-bar bell/heat/ban/patron state, own drunk counter). Actions (`_doSocial`: flirt/kiss/spank/fondle) resolve `_favor(id) − SEV[kind]` into five outcome tiers; `NPC_ROLES` (world.js) caps physical contact for cashiers/mamasans until `bells[room] ≥ 2`. **The bell escalates the whole room while its glow holds** (`_bellLevel` = `bells[room]` when `_bellActive`): `_favor` gains +2/+4/+10 at 1/2/3 rings, so two bells make everyone much friendlier and three bells top-tier every action (hands-on reciprocate); at three, `_addHeat` is amnestied entirely (no repercussions). Heat ≥ 3 → `_kickOut()` (LK Metro bans complex-wide; bans expire after `BAN_TURNS`). Street attempts are negative except the katoey encounter's flirt-back branch. `_doThrowCover` (THROW COVER / NIPPLE COVER / PASTIE [AT <lady>]) is the ceiling-game bit — needs a hostess present and `_favor ≥ 2` for the ammunition, then rolls `_rand()` for stick-time and who it lands on (back on you / another girl = happy; the regular = `patronMiffed` + heat; the mamasan = heat 2). Player-initiated so the dice are fine. `BUY BRA FOR <lady>` (`BRA_PRICE` ฿200, favor ≥ 2) sets `G.soc.bra[id]`, which adds a `braBump` of +2 to net inside `_doSocial` for `fondle` only — one tier warmer; resets nightly with the rest of `G.soc`.

**Hotels** (`_HOTELS`, `G.hotel`: sabai/queenvic/metropole): after Act One, CHECKOUT (`_doCheckout`, in your own room, `nightTurn < 10`) lists the two hotels you're not in (`G.pendingChoice = "checkout"` gates the reply). Rooms: `hotel_room` (Sabai Palms, Naklua), `qv_room` (Queen Vic balcony), `metropole_room` (Metropole tower; `alley` exit = fire stairs into LK Metro). `_doGo` refuses hotel rooms that aren't yours; `_chargeRent` runs at wake — vacation rates 400/700/1300, expat long-stay 270/400/730 — and on failure steps down `_HOTEL_DOWNGRADE` toward the Sabai, then books debt (`G.hotelDebt`, capped `_DEBT_CAP` ฿2000, −1 happy per indebted morning, auto-settled when flush). Debt ≥ ฿800 + entering Stinky/Candy Bar = the once-per-game grace note (`tabSettled` flag). Perks: Sabai = −1 hangover at wake but a ฿300 joiner fee on take-out barfine endings; Queen Vic = `WATCH SOI` (`_doWatchSoi`, +1 happy/night — shares `G.blueDogDay` with the Blue Dog show, deliberately: one "spectator" point a night); Metropole = the robbed ending loses ≤฿1000 and keeps valuables. New vacations and `_goExpat` re-book the Sabai. All hotel perks/fees gate on `_flag("act1Done") && G.stage !== "act1"` — keep new ones consistent.

**Named patrons** (`PATRONS` in world.js — separate from `NPCS`): bar customers with `{name, emoji, age, nat, home, hops, days?, avoids?, desc, dialogue}`. Position is the pure hash `_patronRoom(id)` — hoppers get a hash-picked bar per (vacation, day, hour) until 22:00 (`_patronHour() >= 4`) then settle at `home`; `days` (weekday indices, `G.day % 7`, Monday = 1) keeps a patron home on work nights (David: `[1,5]`); `avoids` lists bars the hop hash skips (Danny: his creditors' bars — keep narrative "he never goes there" claims mechanically true this way). `_patronsHere`/`_findPatron`/`_patronTalk` parallel the NPC pipeline; dialogue schema matches NPC entries but the seen-index book (`G.patronTalk`) resets daily. Rooms list them as "At the rail: …". Patron cross-topics on NPCs (Bert's `danny`, Drew's `angela`) live in normal NPC dialogue.

**The flashlight is social** (`_lightNotice`, on room entry and light-on): silent in dark rooms; hostesses tease by name, barmen point at the lights, street NPCs are amused. In a go-go the house assumes a camera — `_gogoLightWarn` escalates per command (a `mark` flag stops the entry notice and the same command's tick double-counting): warning, warning, `_kickOut()`. Turning it off stands security down and resets the count.

**Quests** live in `QUESTS` (world.js): giver dialogue surfaces offers (`_questOffer` after `_doTalk`), ACCEPT/ABANDON/QUESTS verbs manage `G.quests` states, and `_questTick` (every turn) completes any active quest whose `doneFlag` is set, paying `reward`. `deps` gate offers on other quests being done. The `doneFlag` design is deliberate — quests observe the world rather than owning it, so any mechanic becomes quest-completable with zero quest-specific code; keep it that way. The machinery is ahead of its content (see task list: one quest per district is the next move); grow the schema (stage/clock/favor gates, favor/item rewards) only when a specific quest needs it, and build the future run-your-own-bar expat mechanic AS a dep-chained quest line (premises → license → staff → opening night), not as a new subsystem.

**The phone** (`G.phone`): CONTACT in her bar at favor ≥2 swaps numbers; contacts text unprompted via `_maybeIncomingText` in `_tick` (invites reward showing up that night — checked in `_doGo`; messages can carry money, credited on CHECK MESSAGES); MESSAGE = once-per-night favor charm; SEND <amt> TO <name> is the banking app (favor bump scales with amount). Everything battery-gated.

**Quiz night**: Thursdays (`G.day % 7 === 4`; day 1 = Monday), nightTurn 20–39 (20:00–22:00), at three bars chosen by `_quizBars()` — a pure hash of (vacation, day), deliberately NOT `_rand()` so reading the schedule never advances the dice. Entering a quiz bar mid-window force-starts the quiz (`_quizHere` check in `_doGo`); questions from `QUIZ_POOL` (world.js); one per bar per night via `G.quizPlayed`; QUIT ejects to the street.

Bar mini-games (Connect 4 / Jackpot / pool / killer pool on league nights — `G.day % 3 === 0` at `room.pool` bars) run as a modal `G.game` state: while one is live, `doCommand` routes every input to `_gameInput` and QUIT concedes. Stakes are escrowed up front and paid back ×2 on a win (×3 on a Jackpot); broke players play "for sanuk" (stake 0). **Jackpot has a first-game tutorial**: until `flags.jpLearned`, the hostess (`opp`) walks you through it and *every* roll stops for a manual flip — even a forced single-option one that would normally auto-resolve in `_jpTurn` — so you learn the moves by making them (`_jpTeach` adds a coaching beat the first time you meet a two-way choice and a forced roll; `g.tutorial`/`g.taught` live on the game object). Finishing one full round sets `jpLearned` (survives vacations — `_newVacation` only clears `sabaiSabai`), and from then on forced rolls auto-play and only genuine two-way choices stop for you. Tabletop games are gated on `barType === "beer" | "soi6"`, pool on `room.pool`; `_playOptions()` is the room-aware list all offering surfaces use (bare PLAY fans out in the wheel and lists in prose), and during a live game `_gameVerbs()`/`_c4Choices()`/`_jpChoices()` drive autocomplete and the DROP/FLIP fan-outs.

- `web/js/thai.js` — Thai numbers (สิบเอ็ด/ยี่สิบ irregulars), Thai digits ๐–๙, signs, phrase matching. Pure functions.
- `web/js/news-data.js` — **auto-generated** (`scripts/fetch-news.mjs`, re-baked every 6 h by `.github/workflows/news.yml` from Google News RSS + Frankfurter/er-api FX + Open-Meteo weather; the bake runs the test suite before committing). Classic-script globals `NEWS_FEED`, `FX_RATES` (THB per USD/AUD/GBP/EUR), `WX_NOW` (real Pattaya temp/humidity/WMO code/rain odds), `FOOTY` (ESPN scoreboard — World Cup while it's on, else Premier League; SCORES verb, TV/paper segments, and the patron: every bar's regular gets a lifelong team via pure hash of bar id over the bake's teams, and a win is the ONE football→mechanics crossing — he buys the rail a round, guarded no-bake-no-dice like the rain), `LOTTO` (official GLO draw — LOTTERY verb, TV/paper), `GOLD` (XAU + Thai baht-weight at 96.5%, computed against the same bake's USD rate; paper business corner), and `BTC` (CoinGecko; the laser-eyed patron variant) so no runtime fetch is needed — they surface via WATCH TV / READ PAPER / the WEATHER verb and patron chatter (the FX moan invents a golden-age rate at current × 1.25; the weather moan is always the humidity), and the engine degrades to canned fallbacks when the globals are absent (which is how the vm tests run). **Never gate game logic on headlines or rates** — they change every bake; determinism holds only within a deploy. Weather has exactly ONE sanctioned mechanical crossing: a stormy `WX_NOW.code` (≥95 or 63/65/81/82) *enables* the rainy-season downpour system, but every roll goes through `G.rng` (checked before `_rand()` so a bake-less game consumes no dice — tests and `file://` are byte-identical to before). Downpours (`G.rain` = turns left, started by `_startRain`, 3–8 turns, 30-turn cooldown via `G.lastRain`): street moves are blocked unless the destination passes `_sheltered()` (bar/shop/outlet rooms — diving indoors is allowed), buses and motosai refuse, and the soi-dog dark-streak logic pauses (dogs shelter too). Below the storms, `_wxRainy()` (any rain-family code or rain odds ≥50%) enables **light-rain vignettes** — `_sayDrizzle` in `_tick` (15-turn cooldown via `G.lastDrizzle`) plus a rainy-season patron line — which are atmosphere ONLY: they print, pick their variant by turn parity, and touch no mechanics. Don't add a second weather→mechanics path without the same no-bake-no-dice guard.
- `web/js/world.js` — all rooms, items, NPCs, dialogue, bus/motosai lines. **Pure declarative data**, the source of truth. Dialogue entries are `{req, notFlags, topic, th, rom, text, short, sets, gives}`; first matching entry wins, `_pickDialogue` falls back to the topicless entry on unknown topics. `_deliver` tracks which entries an NPC has already spoken (`G.talked[npcId]` = seen indices, serialized) and on a repeat swaps `text`→`short` and drops the Thai greeting, so re-talking gives the gist not the full spiel; `short` is optional (no field → repeats in full). Author one on any long topicless greeting.
- `web/js/games.js` — bar mini-game logic (Connect 4 + AI, Jackpot shut-the-box dice, abstract 8-ball pool). Pure functions, no `G`, no output; every random decision takes an injected `rnd()`. Loaded between world.js and engine-core.js.
- `web/js/engine-core.js` + `engine-encounters.js` + `engine-play.js` + `engine-systems.js` + `engine-parser.js` — the engine, split into **five classic scripts sharing globals**, loaded in that order after world.js/games.js (index.html and every vm-test load list). `engine-core` declares `G` and the `print`/`speak` hooks and must load first; nothing runs at load beyond const initializers, so functions may live in any part. Each file's header says what it holds; treat them as one engine. Together they own state (`G`), parser, action handlers, systems (battery/darkness/soi dog, money, score, endings). DOM-free at load; all output goes through the `print`/`speak`/`sfx` callbacks injected via `engineInit` (`_engineSfx(name)` is the one-shot sound hook — `_doBell` fires `"bell"`; no-op headless). Random street encounters: scene data in `ENCOUNTERS` (world.js), resolvers in `_ENC` (engine-encounters.js); the RNG is a Lehmer LCG whose state lives in `G.rng`, so rolls serialize with the save and UNDO can't reroll them — tests fire encounters deterministically via `_startEnc(id)`. Interactive encounters set `G.pendingEnc`; the next command is routed to the resolver as the player's snap reaction. Every prompt that leaves `pendingEnc` set must go through `_encPrompt(...[text, cls])` (not bare `_say`), which stashes the prompt lines in `G.encPrompt` so a restore/UNDO can redraw them — `_renderEncounter()` replays the stash. **Restore redraw is centralised in `_renderResume()`** (engine-parser.js): `doCommand` routes input through a chain of *modal input-gating states* — `pendingChoice` (airline / checkout), `G.game`, `pendingEnc`, `pendingFare` — each of which silently swallows commands until answered, so each needs its prompt redrawn on continue/UNDO or the load is blind (the bug class that hit c4, jackpot, and saleng). `_renderResume` is the single dispatcher, mirroring doCommand's gate *priority order*; main.js calls it (not the individual `_renderGame`/`_renderEncounter`) after `_describeRoom`. It also re-surfaces the unread-text count (the "phone buzzed" nudge is a one-shot at arrival, lost on reload). Ambient scene state that a reload must re-establish lives in `_describeRoom` instead: a **downpour** (`G.rain > 0`) re-announces itself there every describe — like darkness does — because rain gates movement, so a silent reload would paint a dry, walkable street and the block that follows would read as a bug. Add a new modal gate to **both** doCommand and `_renderResume`, and route its prompt through a shared helper (`_vacationEndPrompt`/`_checkoutPrompt`/`_farePrompt`/`_encPrompt`/`_renderGame`) so the live prompt, the invalid-answer reprompt, and the resume redraw stay identical. Long scripted tests that walk eligible streets should first mark all of `G.encDone`. **The Zork ledger**: the parser answers the classic-Infocom verb set — DRINK (routes into `_doBuy`), DIAGNOSE (reads the survival stats), AGAIN/G (repeats `_lastCmd` — a module-local on purpose: parser convenience, not game state, so it neither serializes nor concerns a 2D frontend), SMELL/LISTEN (per-`region` tables `_SMELLS`/`_SOUNDS` with a bar override), KILL/HIT/etc. (canon refusal — the piwins are why violence never becomes mechanics), location-aware SWIM/DANCE/SING, and the XYZZY/PLUGH/PRAY/HELLO SAILOR easter eggs. All flavor-only beyond the normal tick. House rule: "I didn't understand that" is a last resort — give a plausible verb a voiced refusal rather than letting it fall through. **QoL/mechanical verbs**: TIME (clock + tonight's barfine/quiz pricing), WAIT <n>/WAIT UNTIL <hour|MIDNIGHT> (fast-forward loop in `_doWait`; stops early on encounters, incoming texts, or night end — each skipped turn still pays full `_tick` costs), TIP <lady> <amt> (฿100+ bumps `soc.drinks` favor, ฿300+ double; small notes are flavor; piwins refuse pre-ride), HAGGLE (peddler-encounter reaction; sets `flags.peddlerDeal` once, ~⅓ off, re-arms `pendingEnc`), WAVE (hails the bus via `_doRideBus`), TRAVEL/GOTO <bar|hotel> (fast travel to any visited bar — `G.visited`, marked in `_describeRoom` — plus your own hotel, which needs no discovering; the room you stand in is never offered; BFS hop count over the exits graph paid in full `_tick`s so it saves keystrokes not time; GO/ENTER with a non-adjacent place name fall through to it, and arrival runs the shared `_arriveAt` — door bans, quiz capture, invitations, encounters — same as walking; free by design for now, revisit pricing later), MAP (`_MAP` ASCII bar-mat map — update it when the street grid changes), WITHDRAW/ATM (mirrors the lobby-ATM hook; stage-aware), plus PHOTO (1% battery), CALL (teaches MESSAGE), SHOWER, CHEERS, SEARCH→examine. **Known names**: `_say` feeds `_learnNames`, which marks a character in `G.known` the first time their name prints (word-boundary, case-sensitive match on the capitalised last word of `name` — "Madam Oy" → Oy; lowercase names like "security" never register); `_topicKnown(topic)` gates every surface that *offers* ask-topics — `engineComplete`'s `ask <npc> about` candidates and term.js's flyout wheel — so nobody's name is suggested before the transcript has mentioned it. Typed ASK is deliberately not gated. Old saves backfill `G.known = {}` in `deserializeGame`.
- `web/js/term.js` / `main.js` — the terminal frontend: scrollback, history, verb chips, autosave (`lbb_save`), continue-prompt (which calls the engine's `_renderResume()` after `_describeRoom` to redraw whatever modal state was gating input — game/encounter/checkout/fare/airline — so none loads invisible), one-level UNDO, music unlock on first gesture, a `#term-send` button so wheel-prefilled commands submit without a keyboard, and the **floating action buttons** in the top-right `#fab-stack` (a flex column so either can appear alone or stacked): `#bell-fab` (shown in bars/go-gos via `_inBar`, tap submits `ring bell`) and `#msg-fab` (shown while `_unreadCount() > 0`, tap submits `check messages` which prints the texts and marks them read, so the glyph self-hides). Both are refreshed together by `_updateFabs()` after every `submit` and at boot — add a new FAB there and to the stack. All persistence lives in `main.js`; the engine is storage-free. **Decoration + the flyout wheel** live here too (presentation, never engine): `decorate()` wraps actionable words in `<b class="kw" data-k data-v>` (npc/patron/bar/item/exit/cmd, plus `thai` — Thai runs tokenised via the vendored tokeniser against WORDS + NPC `th` names so entity names stay whole); tap = `_kwActions(k, v, false)` quick menu (single go/fn action fires instantly — exits walk, plain Thai opens the word card), long-press (500 ms, drag-cancelled) or right-click = the full list including an NPC's live ask-topics. Ask-topics that are somebody's name are hidden until that character is known — the wheel and the input autocomplete both filter through the engine's `_topicKnown` (see the engine bullet). Anonymous staff (NPCs with lowercase names, e.g. "security") only decorate in their own room — elsewhere the word is plain prose, since the absent-name gossip routing ("ask Candy about …") is a dead end for them; capitalised names decorate everywhere on purpose. Actions are `{t, c, go}` commands through the normal submit path (taps echo as typed commands) or `{t, fn}` functions (the translate entry). Collision rule: Thai that is also a game entity gets the wheel with the world's actions first and 🔍 translate riding along; plain Thai goes straight to the modal.

- `web/js/audio.js` — chiptune step sequencer (tracks shared with the trainer; `soi` is a real Sabai Sabai MIDI transcription, `bus` is still an invented Pattaya Pattaya placeholder, now used only as the title/continue-prompt theme). The scheduler plays every track at 75% of its written `bpm` (a single global factor in `_schedule` — full speed felt rushed); keep authoring tracks at their real tempo. `_trackForRoom(roomId, flags)` is the pure track choice — music only on the neon streets (Walking Street `street`, LK Metro/Soi 6 `soi6`) and inside bars; it returns a track name, a set-list array, "surf" (looping-noise sea ambience via `_audio.ambience` — Jomtien and Beach Road streets, interiors excluded), or null for silence, and `_audioForRoom` applies it (`_audio.playlist(names, key)` for arrays). One-shot SFX go through a separate `_sfxBus` (louder `SFX_VOL`, muted alongside music): `_audio.sfx(name)` — `"bell"` is a synthesized metallic **clang** (`_clang`: inharmonic triangle partials for the un-pitched bell body + a band-passed noise strike, played as two quick swings), fired by the engine's `_engineSfx` hook on a bell ring (typed or via the FAB). Tested in tests/js/audio.test.js (the AudioContext stays lazy and unconstructed there) and the bell path in tests/e2e/bell.spec.mjs.

- `web/portraits/` — pixel-art bust portraits (96×96 PNGs, 24×24 grid ×4), one per NPC and patron, generated by `scripts/gen-portraits.py` (pure stdlib; character specs in its `CHARS` dict, composed from a shared part library so the cast has one style). Display is presentation-only in term.js: on the `Here:`/`At the rail:` lines the engine's emoji is swapped for the portrait (`_addAvatars`, a post-`decorate` DOM pass — decorate() output is untouched, so the term.test.js exact-equality assertions stay valid), and a character's flyout wheel gets a `fly-head` portrait header (Thai-name taps resolve through NPC `th` too). ### Designed for the online / shared-world future

**A shared world is the probable long-term destination** (not soon). Structurally this game is already a MUD — rooms/exits/NPCs, text in, prose out, one session per player — so the path is: hosted single-player first (accounts, cloud saves, leaderboards), shared world later. `tests/js/online.test.js` **is the hosting model for the first step**: each player session = one `vm.createContext` holding the game-core files (thai.js, world.js, games.js, and the five engine-*.js parts), commands in via `doCommand`, output out via the injected print callback, persistence via `serializeGame()` blobs. Rules that keep the path open:

1. **The game core (`thai.js`, `world.js`, `games.js`, `engine-*.js`) must stay free of browser and wall-clock APIs** — no `window`/`document`/`localStorage`/`Date`/timers/`fetch`. Time is `G.nightTurn`/`G.day`, never real time. (Currently clean; keep it that way.)
2. **All nondeterminism goes through `G.rng`/`_rand()`** — the only `Math.random` allowed is the initial seed, which a server can inject. Same seed + same command script = identical transcript (tested) — the basis for validation, replay, and anti-cheat.
3. **The single global `G` is fine** — per-session isolation comes from vm contexts, not instance-passing refactors. Don't "fix" the globals.
4. **Output stays in the print callback; persistence stays in `main.js`.** A websocket server replaces term.js/main.js the same way a 2D renderer would.
5. **Calendar checks go through helpers** (`_quizDay`, `_isQuizWindow`, `_leagueTonight`, `_clockStr`, `_weekday`) — in a shared world the clock becomes the server's, and these are the only seams to re-plumb. Never scatter raw `G.day % …` arithmetic.
6. **Shared schedules use pure hashes, not player dice.** `_quizBars()` is the canonical pattern: venue selection derived from (vacation, day) by a stateless hash, so every player would already agree on tonight's quiz bars with zero coordination. Any future "the town decides" feature (events, promotions, which bar the police watch) should follow it.
7. **When adding state, know which side of the wall it's on.** Player-local (most things: money, happy, flags-as-knowledge, favor, heat — heat is *your* reputation and stays per-player even in a shared world) vs world-shared (items lying in rooms, anything a second player could observe or contend for). No refactor needed now — just don't design features that secretly assume the player is alone in the world (e.g. "the only customer tonight" mechanics) without noting it. Contention over the same girls/barfines is a designed feature of the shared world, not a bug to avoid.
8. **The event layer** (structured events instead of prose-only output) is the shared prerequisite for both 2D and multiplayer — still deferred until either actually starts, per the 2D rules below.

### Designed for a future 2D conversion

A 2D version of this game is a live possibility. The text terminal must stay a **disposable frontend**; everything below it must remain frontend-agnostic:

1. **All game logic goes in `engine-*.js`/`world.js`, never in `term.js`/`main.js`.** The terminal renders and persists; it must not know rules.
2. **Every action is its own `_do*` function** (`_doGo`, `_doTalk`, `_doBuy`, …). `doCommand` is only a text parser that maps words onto these — a 2D frontend will call the `_do*` functions directly (tap an exit → `_doGo("n")`) and skip the parser entirely. Never bury game-state changes inside parsing code.
3. **World content stays declarative.** New rooms/items/NPCs are data in `world.js`; presentation-only fields (sprites, positions) can be added later and ignored by the text engine. Don't bake logic into prose.
4. **Game state is one serializable object** (`serializeGame`/`deserializeGame`). Keep it that way — no state in closures, DOM, or module-locals.
5. **Don't build a structured event layer yet.** Prose via `print(text, cls)` is fine until 2D work actually starts; adding events speculatively doubles the output surface. When it starts, the refactor is: handlers emit events, prose becomes one renderer.
6. Time is **turn-based**: `_tick()` runs once per command (battery drain, soi-dog streak). A 2D version should keep ticks on actions/room-entry (roguelike style) rather than going real-time.

### Tests load the real sources via node:vm

`tests/js/*.test.js` read the corresponding `web/js/` file and evaluate it with `vm.runInThisContext` (same realm, so `deepEqual` works). `thai.js`, `world.js`, and the five `engine-*.js` parts are DOM-free at load time and fully testable, including a scripted full playthrough; load the engine parts in order (core → encounters → play → systems → parser) after world.js/games.js. Top-level `const`/`let` from vm-loaded scripts land in the global *lexical* scope — reference them as bare identifiers, not via `globalThis`.

### Browser E2E (tests/e2e/, Playwright)

The node:vm suite never opens `index.html` in a browser — it drives the engine through the print callback. `tests/e2e/smoke.spec.mjs` covers that one gap: it loads `web/index.html` from `file://` in headless Chromium, asserts the boot intro renders, that all five `engine-*.js` parts loaded in order (checks `newGame` from core and `doCommand`/`engineComplete` from parser plus a populated `G`), and that a typed `look` round-trips through the real DOM with no page errors. Deliberately thin — a boot-and-a-command canary for load-order breakage, a missing `<script>` tag, or a boot exception; behavioural depth stays in the vm tests. It runs in CI as its own `e2e` job and **gates the deploy** (`deploy` needs `[test, e2e]`), so a page that fails to boot in a real browser never ships — the tradeoff is that the deploy pipeline now installs Playwright (~1 min). To keep a transient browser hiccup from blocking a release, CI retries the spec twice (`retries: process.env.CI ? 2 : 0` in playwright.config.mjs — up to 3 attempts); locally retries are off so real flakiness surfaces. Run it locally via `npm run test:e2e` after `npm install && npx playwright install chromium`. `node --test` ignores it (`.spec.mjs`, not `.test.js`), so the fast vm suite is unaffected. Since `G` is a lexical global (top-level `let`), the spec's `page.evaluate` reads it as a bare identifier, not `window.G` — the same gotcha as the vm tests.

World-integrity tests enforce: every exit resolves to a real room, all 20 canon bars exist, the patron bench is placed/scheduled sanely, and every flag required by dialogue is settable somewhere (dialogue `sets` plus the engine-set list in `world.test.js` — extend that list when the engine gains new flag-setting actions).
