# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

No build step, no lint, no npm install ‚Äî plain HTML/CSS/JS served as-is.

```sh
# Run all tests (Node 18+)
node --test

# Run a single test file
node --test tests/js/engine.test.js

# Run the game locally
open web/index.html   # works from file://, no network requests
```

Deploy is automatic: any push to `main` runs `.github/workflows/pages.yml` (tests gate the deploy), which publishes `web/` to the `gh-pages` branch. Live at https://soisanuk.github.io/last-baht-bus/.

## Architecture

Zork-style text adventure in the Soi Sanuk / Pattaya universe. Same conventions as the trainer (`/Users/mario/thaicab`): **classic script tags sharing globals, no ES modules, no build step, works from `file://`**. The root `package.json` `"type": "module"` exists only so `node --test` treats test files as ESM. Load order in `index.html` matters (`thai ‚Üí world ‚Üí games ‚Üí engine ‚Üí ‚Ä¶`); `main.js` loads last.

**The game is a staged sandbox.** `G.stage`: `act1` (the wallet quest, day 2 of a 7-day vacation) ‚Üí `vacation` (Act One complete: `_checkAct1` fires once at `hotel_room`, gated in `_doGo` on `hasWallet`, scores the quest, converts score to happiness, and opens the room safe +‡∏ø3000) ‚Üí `expat` (chosen at week's end; endless, +‡∏ø20000 savings; future home of the run-your-own-bar mechanic). `G.over` is never set; the long-game stat is `G.happy` (‡∏™‡∏ô‡∏∏‡∏Å), fed by `_addHappy(n)` hooks across every system. Levels at 0/10/25/50/100; 100 = ‡∏™‡∏ö‡∏≤‡∏¢‡∏™‡∏ö‡∏≤‡∏¢, celebrated once via the `sabaiSabai` flag but non-terminal.

**The clock and the body.** Nights are `NIGHT_TURNS` (100) long, 10 turns/hour from 18:00; `_tick` accrues hunger/thirst, sobers you 1 drink per 20 turns, drains happiness when meters redline, and calls `_endNight(reason)` on collapse (hunger/thirst 100), blackout (drunk ‚â• 9 via `_checkDrunk`), hospital (`G.hurt` ‚â• 3), or dawn. `_endNight` resets dailies (hangover scales next evening's hunger/thirst), advances `G.day`, and respawns at `hotel_room` (or the beach pre-act1); day > 7 outside expat triggers `_endVacation` ‚Üí `G.pendingChoice = "vacation_end"` gate ‚Üí `_newVacation()` (happy resets, `bestHappy` tracks, no lead-in) or `_goExpat()`. Food/water via `FOOD_STALLS`/`_doEat`, plus `room.seven` (one street room per district) selling the cheese toastie/water/charger. BARFINE via `_doBarfine`: favor-gated, priced by `_barfinePrice` (√ó1.5 before 21:00, beer-bar fee waived after midnight except `POPULAR_GIRLS`, flash joints √ó0.75); Soi 6 continues the night, elsewhere `_endNight("barfine")` +10 happy. `_maybeSelfBarfine` (favor ‚â•6, after midnight, once/night) sets `pendingEnc:"selfbf"`. Vacation stage: lobby ATM pays ‡∏ø3000 on first hotel exit per day (`G.atmDay`). Drunk ‚â•5 on lit streets rolls the repeatable `police` pseudo-encounter (30-turn cooldown via `G.lastPolice`); an adjacent `mamaTreat` bar's mamasan rescues at 70%.

Bar social life lives in `G.soc` (per-girl lady-drink counts, per-bar bell/heat/ban/patron state, own drunk counter). Actions (`_doSocial`: flirt/kiss/spank/fondle) resolve `_favor(id) ‚àí SEV[kind]` into five outcome tiers; `NPC_ROLES` (world.js) caps physical contact for cashiers/mamasans until `bells[room] ‚â• 2`. Heat ‚â• 3 ‚Üí `_kickOut()` (LK Metro bans complex-wide; bans expire after `BAN_TURNS`). Street attempts are negative except the katoey encounter's flirt-back branch. `_doThrowCover` (THROW COVER / NIPPLE COVER / PASTIE [AT <lady>]) is the ceiling-game bit ‚Äî needs a hostess present and `_favor ‚â• 2` for the ammunition, then rolls `_rand()` for stick-time and who it lands on (back on you / another girl = happy; the regular = `patronMiffed` + heat; the mamasan = heat 2). Player-initiated so the dice are fine. `BUY BRA FOR <lady>` (`BRA_PRICE` ‡∏ø200, favor ‚â• 2) sets `G.soc.bra[id]`, which adds a `braBump` of +2 to net inside `_doSocial` for `fondle` only ‚Äî one tier warmer; resets nightly with the rest of `G.soc`.

**Quests** live in `QUESTS` (world.js): giver dialogue surfaces offers (`_questOffer` after `_doTalk`), ACCEPT/ABANDON/QUESTS verbs manage `G.quests` states, and `_questTick` (every turn) completes any active quest whose `doneFlag` is set, paying `reward`. `deps` gate offers on other quests being done. The `doneFlag` design is deliberate ‚Äî quests observe the world rather than owning it, so any mechanic becomes quest-completable with zero quest-specific code; keep it that way. The machinery is ahead of its content (see task list: one quest per district is the next move); grow the schema (stage/clock/favor gates, favor/item rewards) only when a specific quest needs it, and build the future run-your-own-bar expat mechanic AS a dep-chained quest line (premises ‚Üí license ‚Üí staff ‚Üí opening night), not as a new subsystem.

**The phone** (`G.phone`): CONTACT in her bar at favor ‚â•2 swaps numbers; contacts text unprompted via `_maybeIncomingText` in `_tick` (invites reward showing up that night ‚Äî checked in `_doGo`; messages can carry money, credited on CHECK MESSAGES); MESSAGE = once-per-night favor charm; SEND <amt> TO <name> is the banking app (favor bump scales with amount). Everything battery-gated.

**Quiz night**: Thursdays (`G.day % 7 === 4`; day 1 = Monday), nightTurn 20‚Äì39 (20:00‚Äì22:00), at three bars chosen by `_quizBars()` ‚Äî a pure hash of (vacation, day), deliberately NOT `_rand()` so reading the schedule never advances the dice. Entering a quiz bar mid-window force-starts the quiz (`_quizHere` check in `_doGo`); questions from `QUIZ_POOL` (world.js); one per bar per night via `G.quizPlayed`; QUIT ejects to the street.

Bar mini-games (Connect 4 / Jackpot / pool / killer pool on league nights ‚Äî `G.day % 3 === 0` at `room.pool` bars) run as a modal `G.game` state: while one is live, `doCommand` routes every input to `_gameInput` and QUIT concedes. Stakes are escrowed up front and paid back √ó2 on a win (√ó3 on a Jackpot); broke players play "for sanuk" (stake 0). Tabletop games are gated on `barType === "beer" | "soi6"`, pool on `room.pool`.

- `web/js/thai.js` ‚Äî Thai numbers (‡∏™‡∏¥‡∏ö‡πÄ‡∏≠‡πá‡∏î/‡∏¢‡∏µ‡πà‡∏™‡∏¥‡∏ö irregulars), Thai digits ‡πê‚Äì‡πô, signs, phrase matching. Pure functions.

Weather has exactly ONE sanctioned mechanical crossing: a stormy `WX_NOW.code` (‚â•95 or 63/65/81/82) *enables* the rainy-season downpour system, but every roll goes through `G.rng` (checked before `_rand()` so a bake-less game consumes no dice ‚Äî tests and `file://` are byte-identical to before). Downpours (`G.rain` = turns left, started by `_startRain`, 3‚Äì8 turns, 30-turn cooldown via `G.lastRain`): street moves are blocked unless the destination passes `_sheltered()` (bar/shop/outlet rooms ‚Äî diving indoors is allowed), buses and motosai refuse, and the soi-dog dark-streak logic pauses (dogs shelter too). Below the storms, `_wxRainy()` (any rain-family code or rain odds ‚â•50%) enables **light-rain vignettes** ‚Äî `_sayDrizzle` in `_tick` (15-turn cooldown via `G.lastDrizzle`) plus a rainy-season patron line ‚Äî which are atmosphere ONLY: they print, pick their variant by turn parity, and touch no mechanics. Don't add a second weather‚Üímechanics path without the same no-bake-no-dice guard.
- `web/js/world.js` ‚Äî all rooms, items, NPCs, dialogue, bus/motosai lines. **Pure declarative data**, the source of truth. Dialogue entries are `{req, notFlags, topic, th, rom, text, short, sets, gives}`; first matching entry wins, `_pickDialogue` falls back to the topicless entry on unknown topics. `_deliver` tracks which entries an NPC has already spoken (`G.talked[npcId]` = seen indices, serialized) and on a repeat swaps `text`‚Üí`short` and drops the Thai greeting, so re-talking gives the gist not the full spiel; `short` is optional (no field ‚Üí repeats in full). Author one on any long topicless greeting.
- `web/js/games.js` ‚Äî bar mini-game logic (Connect 4 + AI, Jackpot shut-the-box dice, abstract 8-ball pool). Pure functions, no `G`, no output; every random decision takes an injected `rnd()`. Loaded between world.js and engine.js.
- `web/js/engine.js` ‚Äî state (`G`), parser, action handlers, systems (battery/darkness/soi dog, money, score, endings). DOM-free at load; all output goes through the `print`/`speak` callbacks injected via `engineInit`. Random street encounters: scene data in `ENCOUNTERS` (world.js), resolvers in `_ENC` (engine.js); the RNG is a Lehmer LCG whose state lives in `G.rng`, so rolls serialize with the save and UNDO can't reroll them ‚Äî tests fire encounters deterministically via `_startEnc(id)`. Interactive encounters set `G.pendingEnc`; the next command is routed to the resolver as the player's snap reaction. Long scripted tests that walk eligible streets should first mark all of `G.encDone`. **The Zork ledger**: the parser answers the classic-Infocom verb set ‚Äî DRINK (routes into `_doBuy`), DIAGNOSE (reads the survival stats), AGAIN/G (repeats `_lastCmd` ‚Äî a module-local on purpose: parser convenience, not game state, so it neither serializes nor concerns a 2D frontend), SMELL/LISTEN (per-`region` tables `_SMELLS`/`_SOUNDS` with a bar override), KILL/HIT/etc. (canon refusal ‚Äî the piwins are why violence never becomes mechanics), location-aware SWIM/DANCE/SING, and the XYZZY/PLUGH/PRAY/HELLO SAILOR easter eggs. All flavor-only beyond the normal tick. House rule: "I didn't understand that" is a last resort ‚Äî give a plausible verb a voiced refusal rather than letting it fall through. **QoL/mechanical verbs**: TIME (clock + tonight's barfine/quiz pricing), WAIT <n>/WAIT UNTIL <hour|MIDNIGHT> (fast-forward loop in `_doWait`; stops early on encounters, incoming texts, or night end ‚Äî each skipped turn still pays full `_tick` costs), TIP <lady> <amt> (‡∏ø100+ bumps `soc.drinks` favor, ‡∏ø300+ double; small notes are flavor; piwins refuse pre-ride), HAGGLE (peddler-encounter reaction; sets `flags.peddlerDeal` once, ~‚Öì off, re-arms `pendingEnc`), WAVE (hails the bus via `_doRideBus`), MAP (`_MAP` ASCII bar-mat map ‚Äî update it when the street grid changes), WITHDRAW/ATM (mirrors the lobby-ATM hook; stage-aware), plus PHOTO (1% battery), CALL (teaches MESSAGE), SHOWER, CHEERS, SEARCH‚Üíexamine.

### Designed for the online / shared-world future

**A shared world is the probable long-term destination** (not soon). Structurally this game is already a MUD ‚Äî rooms/exits/NPCs, text in, prose out, one session per player ‚Äî so the path is: hosted single-player first (accounts, cloud saves, leaderboards), shared world later. `tests/js/online.test.js` **is the hosting model for the first step**: each player session = one `vm.createContext` holding the four core files, commands in via `doCommand`, output out via the injected print callback, persistence via `serializeGame()` blobs. Rules that keep the path open:

1. **The game core (`thai.js`, `world.js`, `games.js`, `engine.js`) must stay free of browser and wall-clock APIs** ‚Äî no `window`/`document`/`localStorage`/`Date`/timers/`fetch`. Time is `G.nightTurn`/`G.day`, never real time. (Currently clean; keep it that way.)
2. **All nondeterminism goes through `G.rng`/`_rand()`** ‚Äî the only `Math.random` allowed is the initial seed, which a server can inject. Same seed + same command script = identical transcript (tested) ‚Äî the basis for validation, replay, and anti-cheat.
3. **The single global `G` is fine** ‚Äî per-session isolation comes from vm contexts, not instance-passing refactors. Don't "fix" the globals.
4. **Output stays in the print callback; persistence stays in `main.js`.** A websocket server replaces term.js/main.js the same way a 2D renderer would.
5. **Calendar checks go through helpers** (`_quizDay`, `_isQuizWindow`, `_leagueTonight`, `_clockStr`, `_weekday`) ‚Äî in a shared world the clock becomes the server's, and these are the only seams to re-plumb. Never scatter raw `G.day % ‚Ä¶` arithmetic.
6. **Shared schedules use pure hashes, not player dice.** `_quizBars()` is the canonical pattern: venue selection derived from (vacation, day) by a stateless hash, so every player would already agree on tonight's quiz bars with zero coordination. Any future "the town decides" feature (events, promotions, which bar the police watch) should follow it.
7. **When adding state, know which side of the wall it's on.** Player-local (most things: money, happy, flags-as-knowledge, favor, heat ‚Äî heat is *your* reputation and stays per-player even in a shared world) vs world-shared (items lying in rooms, anything a second player could observe or contend for). No refactor needed now ‚Äî just don't design features that secretly assume the player is alone in the world (e.g. "the only customer tonight" mechanics) without noting it. Contention over the same girls/barfines is a designed feature of the shared world, not a bug to avoid.
8. **The event layer** (structured events instead of prose-only output) is the shared prerequisite for both 2D and multiplayer ‚Äî still deferred until either actually starts, per the 2D rules below.

### Designed for a future 2D conversion

A 2D version of this game is a live possibility. The text terminal must stay a **disposable frontend**; everything below it must remain frontend-agnostic:

1. **All game logic goes in `engine.js`/`world.js`, never in `term.js`/`main.js`.** The terminal renders and persists; it must not know rules.
2. **Every action is its own `_do*` function** (`_doGo`, `_doTalk`, `_doBuy`, ‚Ä¶). `doCommand` is only a text parser that maps words onto these ‚Äî a 2D frontend will call the `_do*` functions directly (tap an exit ‚Üí `_doGo("n")`) and skip the parser entirely. Never bury game-state changes inside parsing code.
3. **World content stays declarative.** New rooms/items/NPCs are data in `world.js`; presentation-only fields (sprites, positions) can be added later and ignored by the text engine. Don't bake logic into prose.
4. **Game state is one serializable object** (`serializeGame`/`deserializeGame`). Keep it that way ‚Äî no state in closures, DOM, or module-locals.
5. **Don't build a structured event layer yet.** Prose via `print(text, cls)` is fine until 2D work actually starts; adding events speculatively doubles the output surface. When it starts, the refactor is: handlers emit events, prose becomes one renderer.
6. Time is **turn-based**: `_tick()` runs once per command (battery drain, soi-dog streak). A 2D version should keep ticks on actions/room-entry (roguelike style) rather than going real-time.

### Tests load the real sources via node:vm

`tests/js/*.test.js` read the corresponding `web/js/` file and evaluate it with `vm.runInThisContext` (same realm, so `deepEqual` works). `thai.js`, `world.js`, `engine.js` are DOM-free at load time and fully testable, including a scripted full playthrough. Top-level `const`/`let` from vm-loaded scripts land in the global *lexical* scope ‚Äî reference them as bare identifiers, not via `globalThis`.

World-integrity tests enforce: every exit resolves to a real room, all 15 canon bars exist, and every flag required by dialogue is settable somewhere (dialogue `sets` plus the engine-set list in `world.test.js` ‚Äî extend that list when the engine gains new flag-setting actions).
